**Article 12**

**Implementation of Automated Event Logging Throughout System Lifetime**

Gas Safety Insight incorporates an automated comprehensive logging framework embedded directly into its core data processing and decision-making modules. This framework captures detailed event records continuously from the system’s initialization through decommissioning to ensure full traceability across its lifecycle. Logging is triggered by both routine operational checkpoints and system-triggered events, including model inference executions and sensor data preprocess stages, recorded with precise timestamps and contextual metadata.

The logging subsystem integrates with the hybrid model architecture, capturing granular inputs and outputs from the Gradient Boosted Decision Trees (GBDT) and encoder-only Transformer components. For each inference cycle, sensor fusion inputs—including pressure levels, flow rates, and temperature readings—are logged alongside the temporal and contextual embeddings generated by the Transformer backbone. Furthermore, the system logs internal confidence scores and anomaly detection flags produced by the GBDT ensemble. All logs are stored in an encrypted, tamper-evident format compliant with industry-standard secure audit trail protocols, facilitating forensic integrity and compliance validation over time.

**Identification and Recording of Risk-Related Situations and Substantial Modifications**

To address potential risks as defined under Article 79(1), the logging mechanism is explicitly tuned to capture triggers indicative of safety hazards or operational deviations. These include but are not limited to sensor readings exceeding risk thresholds, abnormal temporal patterns detected by the Transformer’s attention maps, and GBDT-predicted hazard classifications. Each such event is documented with detailed diagnostic data, including sensor provenance and model component responses, enabling root cause analysis.

Substantial modifications of the AI system, such as model retraining cycles or hyperparameter changes, are captured as distinct log entries. Versioning metadata for both model components (GBDT and Transformer parameters) and upstream data preprocessing pipelines are automatically recorded, including timestamps of deployment and prior validation metrics. This systematic recording ensures traceability not only of outputs but also of the evolving internal system state and configuration, supporting robust lifecycle governance and auditability.

**Support for Post-Market Monitoring Activities**

Gas Safety Insight’s logging architecture facilitates post-market monitoring by systematically recording operational context and anomaly detection history crucial for ongoing safety assessment. Logs include real-time operational metrics and longitudinal summaries of detected risks, enabling trend analysis over extended periods. The logging system interfaces with external post-market monitoring platforms through secured APIs, providing continuous data streams and alert event logs to assist network operators and regulatory assessors.

Additionally, the system generates periodic diagnostic reports aggregating event logs, detection frequencies, and anomaly classification outcomes to support maintenance planning and performance benchmarking. These logs underpin data-driven maintenance cycles and safety audits, directly contributing to compliance monitoring as per Article 72 requirements.

**Continuous Monitoring of System Operation and Functionality**

Functionality monitoring is realized through dedicated telemetry logged throughout normal and edge operational scenarios. System health indicators—such as data input integrity checks, model latency metrics, and sensor data pre-validation results—are captured to detect malfunctions or performance degradation. The logging includes flagged instances of input irregularities (e.g., missing data or sensor faults) that could impact inference accuracy.

Anomaly detection outcomes feed directly into operating condition logs, with both raw sensor inputs and model decision traces preserved. This approach allows retrospective verification of system behavior under normal and exceptional circumstances, enabling detailed performance analyses consistent with monitoring expectations outlined in Article 26(5). The logging infrastructure supports secure access for authorized compliance verifiers, permitting real-time and historical inspection of system operation parameters.
