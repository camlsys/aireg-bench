**Article 12**

**Logging Architecture and Event Capture Framework**  
The Guardian Signal Controller is equipped with an integrated event-logging framework designed to record operational data continuously throughout the system’s full lifecycle. This logging subsystem is implemented as an embedded, tamper-resistant module residing within the edge computing units located at traffic intersections. It captures and timestamps a comprehensive set of system events including input data snapshots, intermediate feature extraction outputs, classifier decision states, and control signal commands transmitted to traffic lights. Specifically, raw sensor feeds and video frames are periodically downsampled and retained in encrypted buffers linked to triggered anomaly detection events to balance data fidelity and storage constraints. Metadata elements include system software version, model checkpoints (CNN weights and Random Forest parameters), hardware status reports, and network connectivity logs. This design supports automated and uninterrupted log generation without requiring manual intervention, facilitating auditability over the system’s operational lifespan.

**Captured Events Relevant to Risk Identification and Modification Detection**  
The logging capabilities prioritize events instrumental for detecting scenarios that may elevate risk or indicate substantial system changes. These include:  
- Alerts raised by the anomaly detection module when traffic patterns deviate from learned baselines (e.g., potential red-light violations or pedestrian conflicts), accompanied by corresponding video feature maps and classification confidences;  
- System warnings triggered by classification uncertainty thresholds or sensor malfunctions that could compromise decision accuracy;  
- Records of model retraining cycles or parameter updates initiated remotely or on-site, including versioning information and performance metrics observed during validation tests;  
- Manual overrides or emergency control inputs logged along with operator identifiers and temporal context, to account for exceptional interventions affecting signal control logic.  
This event set enables retrospective identification of risk instances and tracks modifications that substantively impact AI model behavior or system safety boundaries.

**Support for Post-Market Monitoring Processes**  
Post-market surveillance is supported through structured export and secure transmission of aggregated, anonymized logs to Aegis Mobility Technologies’ central monitoring platform. The system architecture incorporates configurable thresholds for event selection, facilitating tailored reporting consistent with evolving operational feedback and regulatory requirements. Logs supporting post-market analysis include performance degradation indicators, event recurrence statistics, and environment perturbation records (e.g., adverse weather conditions impacting sensor reliability). Data integrity is ensured by employing cryptographic hashing and chain-of-custody metadata, facilitating tamper-evident audit trails. Moreover, diagnostics data streams inform periodic safety reviews and underpin continuous improvement workflows aligned with Article 72 mandates.

**Operational Oversight and Real-Time Monitoring Functions**  
To enable effective monitoring of ongoing operation as required under Article 26(5), the system maintains a real-time dashboard interface accessible to authorized traffic management personnel. This interface presents live status indicators derived from event logs, such as classifier confidence distributions, detected anomalies at intersections, sensor health statistics, and communication latency metrics. Behind the scenes, an alert management engine ingests log data to trigger tiered alerts escalating from informatory notices to critical warnings based on predefined risk models. These alerts prompt immediate investigation or intervention, supporting responsive traffic control optimization and safety assurance. The continuous feedback loop between logging outputs and operational monitors ensures that the AI system’s functioning remains transparent and traceable to stakeholders responsible for urban traffic governance.
